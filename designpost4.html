How Does The Code Impact The UX and UI of This Website-

I have used HTML, CSS, and JavaScript for this website. My folder structure does reflect my content through Hierarchical Information Structure. All three coding languages separate in layers. Data layer represents the HTML code, the logic layer represents the JavaScript code, and the presentation layer represents the CSS code. My naming conventions are modular and scalable as they are usually my headings are sized at 30pixels- 15pixels the paragraph text is usually sized at 15pixels-12pixels. The naming conventions are linked to the name of the section. I have tried to use consistent tag containers and styling conventions throughout the website in terms of positioning, font sizes, margin sizes etc.

My markup is providing enough information for a user to use browser to access my website. The believe the interface elements and the content are expressed fully and correctly allowing users easy navigation throughout the website. The usability is portrayed through accessibility, clarity, and responsive design. Accessibility can be improved through more media queries.

The JavaScript codes do express interaction points through the buttons, the links, and the wireframe gallery. The array type and the convert from HTML to JavaScript innerHTML are repeated. Unfortunately, I couldn’t get all the code to separate and consolidate. Yes, I have made various JavaScript’s that link through the HTML. I have tried my best in async requests to decrease redownloading and to improve the overall performance of the UX and UI. Sadly, I was still struggling with DOM JavaScript up until the end.

The CSS scripts are linked to the HTML preventing in-line styles and I tried my best to maintain consistent styling methods throughout the website. The code is modular and maintainable. I have adapted responsive layouts of screen sizes in response to the desktop size (1440pixels), tablet size (768pixels) and mobile phone medium-size (375pixels). I have implemented my HTML and CSS codes in a way that provides effective impact in the UI and UX systems functionally and intuitively.

Media Queries-

From assignment 1, I learnt that the media query sizes I used were not utilised in the correct manner. I decided that I would use 4 different screen size resolutions: 1500px, 800px, 500px and 400px. The 850px were part of the codes from assignment 1, hence why that code is still present in the CSS scripts. The 1500px screen resolution should enable a person using a laptop of (1440pixels) screen size to view the website without any worries. The 800px screen resolution can accommodate a tablet size of (768pixels) and the (400pixels) screen resolution will be used to size down a mobile phone screen of (375pixels-320pixels) allowing for easy on the go access. These screen size resolutions are beneficial to the user as they can experience this website wherever they are on any device that accommodates these sizes creating a friendly user experience.
