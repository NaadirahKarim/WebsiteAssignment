The UI relation to the interactivity definition from last week From last week, I know interactivity is a powerful tool for UI design. I may repeat what is already known but to me interactivity and UI go hand in hand to provide engagement for the user to be satisfied. The interface is an exchange barrier between a user and the system which authorizes and characterizes interaction for a productive experience. Interactivity within the UI requires balance, too much can be overwhelming but too little can be frustrating which is one of the cons of interaction and convention. Modern-day users all around the world expect the UI of any app or website to be quick and responsive with effective feedback. UI design is used to guide and advise user experience in development and improvement. Thus, accessibility and useability are important fundamentals for UI and UX.

My Reflection on "The Basic Principles of User Interface Design" and "Gibson - 2004 - Alternative Approaches to Interface Technology” readings After reading both "The Basic Principles of User Interface Design.pdf" and "Gibson - 2004 - Alternative Approaches to Interface Technology.pdf, there was a clear understanding that both have different perspectives on user interface design, yet they both have the same focus on improving user experience. The former article focuses on the principles of design that are essential elements of design, whilst the other article is based on alternative approaches to UI technology which are clear and available to understand. Both articles provide clear perspectives that the UI should be designed by the user’s satisfaction. Due to technological advancements new usability challenges and new interface designs are recognized. Thus, both articles provide valuable insights into the complexity and dynamics of UI design as well as how the basic principles and technology can make interface designs enjoyable for users.
